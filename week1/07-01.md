> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

###### 1. 오늘 완료한 것
1. HTTP request의 첫 라인 파서 클래스 작성.
2. 라인 파서 클래스의 테스트 케이스를 작성하고 설계

###### 2. 오늘 가장 의미있었던 기술적 판단
- 문제 상황 : 유틸 파서클래스를 TDD로 개발하기 위해서 테스트 케이스를 작성하는 도중 연속 공백으로 분리된 라인을 예외처리할 것인지 아닌지를 판단하는 문제
- 내가 내린 판단: 연속 공백 예외 처리 코드는 제외하기로 결정.
- 그렇게 판단한 이유: 연속 공백 라인은 정상적인 클라이언트에서 오는 경우는 없음.
- 고려했지만 선택하지 않은 대안: 저장된 배열의 method, target, version을 순회하며 정상적인 입력인지 일일이 검사한다.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향: 배열 순회 과정이 생략되어 파서 생성 속도가 더 빨라진다.

###### 3. 가장 막혔던 문제와 해결 과정
- 문제 상황 또는 증상: ./index.html을 FILE객체로 생성하는 중에 파일 경로를 정확하게 읽지 못하는 문제
- 처음 추측한 원인: 상대 경로를 인식하지 못한다고 생각하였음. 왜냐하면 예제에 있는 경로가 잘못되었다고 생각했음.
- 원인을 확인하기 위해 시도한 방법: jdk의 nio 라이브러리 대신에 절대 경로를 사용하는 것을 고려함.
- 실제 원인:File의 상대 경로는 소스 코드나 리소스가 위치한 곳이 아니라, JVM이 실행된 현재 작업 디렉토리(user.dir)를 기준으로 해석된다. 리소스 파일을 src/main/resources/static에 두었는데, resources 아래의 파일은 빌드 시 build/resources/main으로 복사되어 클래스패스로 접근하도록 되어 있다. 즉 File이 바라보는 파일시스템 경로와, 리소스가 실제로 놓이는 위치가 서로 다른 체계였다. 처음 시도한 ./static/...이나 ./src/resources/static/...은 이 구조와 맞지 않아 존재하지 않는 경로를 가리켰고, 그래서 파일을 찾지 못했다.
- 해결 방법: 경로를 실제 구조에 맞게 ./src/main/resources/static + target으로 수정하여 동작을 확인했다. 다만 이 방식은 실행 중에 소스 트리를 직접 읽는 것이라, 실행 위치가 바뀌거나 JAR로 배포하면 src 폴더가 사라져 깨질 수 있다는 한계가 있다. 더 견고한 방식은 File 대신 getClass().getClassLoader().getResourceAsStream("static/...")으로 클래스패스에서 읽는 것이며, 이 경우 빌드된 리소스를 참조하므로 배포 환경에서도 안전하다.
- 다음에 비슷한 문제가 생기면 먼저 확인할 것: 상대 경로가 안 맞을 때는 추측하기 전에 System.getProperty("user.dir")로 현재 작업 디렉토리를 먼저 찍어서 기준점을 확인한다.

###### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지: HTTP헤더의 첫 라인을 파싱하는 유틸 클래스를 작성할 때, 테스트 케이스를 먼저 작성하고 개발에 들어간 것.
- 그 접근이 도움이 된 이유: 테스트 케이스를 작성하는 중에, HTTP 응답이 실제로 어떻게 망가질 수 있는 지 공부하는 기회가 되었음.

###### 5. 더 개선할 점
- 비효율적이었던 접근/놓치거나 부족했던 점: FILE 객체를 생성할 때, 클래스 루트가 아니라 하드 코딩된 절대 경로로 작성하여, 유지보수 측면에서 볼때 확장성이 부족하다고 느꼈다.
- 그렇게 된 원인: java로 개발을 처음하다 보니 잘못된 것은 느꼈지만 구현을 할 능력과 지식적으로 부족한 것 같다.
- 다음에는 다르게 해볼 구체적인 행동: 현재 404 응답과, 200응답을 보내는 클래스가 두개로 각각 되어 있는데, 이 부분도 확장성 측면에서 볼 때 개선의 여지가 있다. 요구사항이 Concurrent 라이브러리를 사용하라고 되어있었는데, 이 부분에 대해 학습이 더 필요하다.
