# be-DailyJob 

> 이 회고는 작업량을 보고하기 위한 문서가 아니라, 
> 오늘 내가 어떤 판단을 했고 어떤 개발 행동을 반복하거나 바꿔야 하는지 확인하기 위한 기록입니다. 
> 각 항목은 길게 쓰기보다, 사실/판단/근거/다음 행동이 드러나게 구체적으로 작성하세요.

### 1. 오늘 완료한 것
- 오늘 실제로 끝낸 기능/수정/학습:
  - 통합된 유틸 파서 테스트 케이스 작성
  - request line parser 의 target 파싱에 URI 라이브러리 사용하는 것으로 리팩토링
  - 로그인 후 회원의 이름을 로그인/회원가입 버튼 대신 나타내는 기능 추가.
  - 로그인 후 회원가입/로그인 페이지 접근 시 메인 페이지로 리다이렉트하는 기능 추가.
  - 로그 아웃 시도 중 쿠키가 없는 경우에도 동작하도록 변경

### 2. 오늘 가장 의미있었던 기술적 판단
- 내가 내린 판단:
  - request line target의 파싱을 URI에게 위임한 것.
- 그렇게 판단한 이유:
  - 이미 해당 메서드에 URI 라이브러리를 사용하고 있었는데, path 파싱에만 URI를 사용하지 않는 것은 모순된다고 생각함.
- 고려했지만 선택하지 않은 대안:
  - 직접 파싱로직을 작성하는 것.
- 이 판단이 코드 품질/테스트/유지보수에 미친 영향:
  - 신뢰도 있는 라이브러리의 도입으로 path의 형식 검사 로직이 더 단단해짐.

### 4. 오늘 잘한 행동(개발과 관련된 것)
- 어떤 행동이었는지:
  - 하나의 클래스를 작성하고 나면 테스트 케이스를 claude code /plan 으로 요청하는 것.
- 그 접근이 도움이 된 이유:
  - 내가 놓친 엣지 케이스를 검토해보는 시간을 단축할 수 있음.
- 다음에도 반복하고 싶은 점:
  - /skills 를 통해서 하나의 클래스 작성이 끝나면 자동으로 테스트를 진행하는 것.
